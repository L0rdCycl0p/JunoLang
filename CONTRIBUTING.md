# Contributing

First of all, thank you for your interest in contributing to this project!

Whether you're fixing a typo, reporting a bug, improving the documentation, or implementing a new feature, every contribution is appreciated.

## Before You Start

* Read the `README.md`.
* Check whether an issue already exists for your idea or bug.
* If you're planning a large change, please open an issue first to discuss it.

## Contributor Profile

On your **first contribution**, you **must** create your contributor profile in the `CONTRIBUTORS/` directory.

The filename **must** match your contributor ID.

Example:

```text
CONTRIBUTORS/johndoe.yml
```

Please follow the schema described in `CONTRIBUTORS/README.md`.

## Pull Requests

Before opening a pull request, make sure that:

* Your code builds successfully.
* Existing tests continue to pass.
* New functionality includes tests whenever practical.
* Documentation has been updated if necessary.
* Your changes are focused on a single purpose.

## Coding Style

* Write clean and readable code.
* Prefer descriptive names over abbreviations.
* Keep functions and modules focused on a single responsibility.
* Follow the existing style of the project.
* Avoid unnecessary dependencies.

## Commits

Please write clear commit messages.

Examples:

```text
feat(parser): add JSON parser

fix(core): prevent null pointer dereference

docs: improve installation guide

refactor(cli): simplify argument parsing
```

## Respect the Project

Please:

* Be respectful to other contributors.
* Accept constructive feedback.
* Keep discussions professional.
* Focus on improving the project.
